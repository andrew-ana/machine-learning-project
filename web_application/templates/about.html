Project Component Description
Data Sources

Our primary source of data was On-Time performance data on flights departing from US cities as published by the Bureau of Transportation Statistics. This data was supplemented with BTS information on the column headings.

Data Cleaning The data was processed in Python. The number of categories was reduced to those columns that would be predictive of on time arrival such as departure city, arrival city, airline, and datetime. Month, day of the week, and time of day columns were created as well as a target response column for late or on time arrival. In addition, location data on on the airports including city, latitude, and longitude was merged with the dataframe for later map creation.
Data Base Our first attempt at creating a database was with loading the table directly into SQL. Due to the large file sizes that were involved (16 million rows), however, this was not functional. Instead, the data was loaded into AWS-RDS and a host connection was created with PostgresSQL. This database was queried using SQL Alchemy. Limitations in processing capabilities often required us to use subsamples of the table for analysis.
Visualizations Several data visualizations were created using Plotly in Javascript using a 10% sample of 2019 data. These included bar graphs of delays by carrier, cause, time of day, and worst routes. A line graph was also created for delay by day of the week. Two geojsons were created from a 200k row sample from the final data frame: one for departure airports and one for arrival airports. These were use to create two bubble maps using Leaflet that depict the average flight delay by airport.
Machine Learning To predict the probability of late arrival we built a neural network in Keras. Prior to training, categorical features were dummified in pandas and the target was label endcoded in Keras. We used a sequential network with dense layers. The input had 790 dimensions, the hidden layer had 40 units (activation = 'relu') and the output layer had two units (activation = 'softmax'). We used the Adam optimizer with a learning rate of 0.001, with 'binary_crossentropy' as the loss function, and 'AUC' as our test score. The model was trained over 10 epochs with the default batch size of 32. For comparison, h2o AutoML was also tested on a sample of the data set. The best results achieved when training with AutoML was with the Stacked Ensemble (AUC = 0.63). This underperformed the neural network described previously.
Flask App and Hosting The backend is served with Flask and hosted on Heroku.
